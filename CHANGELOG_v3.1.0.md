# CHANGELOG - WorldBet League Phase 3

## Magic Links + In-App Notification Inbox + Browser Notifications

### Version 3.1.0 - 2026-05-11 (Updated: In-App Inbox System)

#### 🎯 Major Update: In-App Notification Inbox + Browser Notifications API

**Architectural Shift**:

- **v3.0.0**: Push notifications sent to external service (SendGrid/AWS)
- **v3.1.0**: Notifications stored in-app + sent to browser via Notification API

#### ⚠️ Removed (Breaking Changes)

- ❌ SendGrid integration (was sending emails for magic links)
- ❌ AWS SES integration (was sending push notifications)
- ❌ External push service dependency
- ❌ Legacy PUSH_SUBSCRIPTION table (now uses browser API)

#### ✨ Added (v3.1.0)

1. **NotificationInboxService** - Complete in-app notification management
   - 8 core methods for CRUD operations
   - Real-time subscriptions via Supabase realtime
   - Bulk notification support
   - Pagination and filtering

2. **NotificationInboxComponent** - Beautiful UI for notification bandeja
   - Angular standalone component
   - Signal-based state (reactive)
   - OnPush change detection
   - Badge showing unread count
   - Mark as read (single/all), archive, delete

3. **NOTIFICATION_INBOX Table** - Persistent notification storage
   - Fields: id, user_id, type, title, body, payload, read_at, is_archived, priority
   - Audit logging via trigger
   - Soft delete compliance

4. **Browser Notifications API Integration**
   - Uses native W3C Notification API
   - P-256 ECDSA key support
   - No external library dependency

#### 🗄️ Database Changes (v3.1.0)

**New Table: NOTIFICATION_INBOX**

```sql
CREATE TABLE NOTIFICATION_INBOX (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id BIGINT NOT NULL REFERENCES USER(id) ON DELETE CASCADE,
  type VARCHAR(50) NOT NULL,
  title VARCHAR(255) NOT NULL,
  body TEXT,
  payload JSONB,
  read_at TIMESTAMPTZ,
  is_archived BOOLEAN DEFAULT FALSE,
  priority VARCHAR(20) DEFAULT 'normal',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  is_deleted BOOLEAN DEFAULT FALSE,
  deleted_at TIMESTAMPTZ,
  deleted_by BIGINT REFERENCES USER(id)
);

CREATE INDEX idx_notification_inbox_user_id ON NOTIFICATION_INBOX(user_id);
CREATE INDEX idx_notification_inbox_created_at ON NOTIFICATION_INBOX(created_at DESC);
CREATE INDEX idx_notification_inbox_is_read ON NOTIFICATION_INBOX(read_at IS NULL);
CREATE INDEX idx_notification_inbox_deleted ON NOTIFICATION_INBOX(is_deleted);
```

**Supported Notification Types**:

- `prediction_locked` - Predictions closed (15min before match)
- `match_reminder` - Match starting soon
- `league_update` - League status change
- `league_created` - New league created
- `invitation_received` - New league invite
- `invitation_accepted` - Someone joined your league
- `result_posted` - Match result available
- `user_added_to_league` - You were added to league

**Priority Levels**:

- `low` - Informational (blue)
- `normal` - Standard (gray)
- `high` - Important (orange)
- `urgent` - Critical action needed (red)

**Removed Table**:

- ❌ PUSH_SUBSCRIPTION - Replaced by browser Notification API

#### 🔄 Notification Flow (v3.1.0)

```
Event Triggered (match start, invitation, etc)
         ↓
[createInboxNotification()]
         ↓
INSERT INTO NOTIFICATION_INBOX (database persisted)
         ↓
[sendBrowserNotification()] async/non-blocking
         ↓
Browser Notification API (real-time alert)
         ↓
INSERT INTO AUDIT_LOG (audit trail)
         ↓
User sees:
  1. In-app notification in component
  2. Browser alert (if permitted)
  3. Badge shows unread count
```

#### 💻 Code Examples (v3.1.0)

**Sending a Notification**:

```typescript
// In league-creation.service.ts
await this.notificationInboxService.storeNotification(
  userId,
  'prediction_locked',
  '🔒 Predicciones cerradas',
  'Las predicciones para Argentina vs Mexico cerraron en 15 min',
  {
    leagueId: 1,
    matchId: 42,
    teamsInfo: { home: 'Argentina', away: 'Mexico' },
    actionUrl: '/league/1/standings',
  },
);
```

**Sending to Multiple Users**:

```typescript
const userIds = [1, 2, 3, 4, 5];
for (const userId of userIds) {
  await this.notificationInboxService.storeNotification(
    userId,
    'prediction_locked',
    title,
    body,
    payload,
  );
}
```

**Using in Component**:

```typescript
export class MyComponent {
  unreadCount = signal(0);

  constructor(private notif: NotificationInboxService) {}

  async ngOnInit() {
    // Get current count
    const count = await this.notif.getUnreadCount(userId);
    this.unreadCount.set(count);

    // Subscribe to changes
    this.notif.subscribeToNewNotifications(userId).subscribe((newNotif) => {
      this.unreadCount.update((c) => c + 1);
    });
  }
}
```
